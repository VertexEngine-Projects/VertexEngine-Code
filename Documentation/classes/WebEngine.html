VertexEngine WebEngine

Description: The VertexEngine WebEngine module provides an embedded web browser powered by QWebEngine. It supports custom browser profiles, JavaScript-to-Python communication, and secure page loading. This system is ideal for launchers, editors, dashboards, and web-powered UI tools. use pip install VertexEngine-WebEngine to use it.

WebEngine

Description: The main web view widget. It combines a custom browser profile, page controller, web channel, and JavaScript bridge into a single, easy-to-use component.

Features

  • Embedded Chromium-based browser
  • Custom user agent and storage paths
  • JavaScript ↔ Python communication
  • Automatic WebChannel setup

Example: Usage

from VertexEngine.WebView import WebEngine

web = WebEngine()
web.load_url("https://example.com")
web.show()

Functions

  • load_url(url: str)
    Loads the given URL into the web view.

Returns

A fully configured web browser widget.

WebProfile

Description: A custom browser profile that controls cookies, cache, storage, and the user agent. Each WebEngine instance uses this profile.

Behavior

  • Custom user agent (VertexWebEngine/1.0)
  • Persistent cookies enabled
  • Disk-based cache and storage

Storage Paths

  • webcache/ – HTTP cache
  • webstorage/ – Local & persistent storage

Returns

A reusable web browser profile.

WebPage

Description: A custom page controller connected to the WebProfile and WebChannel. Handles page-level behavior and JavaScript integration.

Note: This class is internally managed by WebEngine and does not need to be created manually.

Returns

The active web page instance.

JSBridge

Description: A bridge object exposed to JavaScript via QWebChannel. Allows JavaScript code to call Python methods directly.

Available Methods

  • log(message: str)
    Sends a message from JavaScript to Python and prints it to the console.

JavaScript Example

bridge.log("Hello from JavaScript!")

Python Output

[JS → Python] Hello from JavaScript!

Returns

Cross-language communication between JavaScript and Python.

JavaScript ↔ Python Communication

The WebEngine automatically registers a JavaScript object named bridge using QWebChannel.

How It Works

  1. Python creates a JSBridge object
  2. The object is registered as bridge
  3. JavaScript can call exposed Python slots

Use Cases

  • Web-based UI controls
  • Debug consoles
  • Editor dashboards
  • Hybrid desktop/web apps

⚠️ Important Notes

  • This system is not intended for games
  • Designed for tools, editors, and UI-heavy applications
  • Requires PyQt6-WebEngine
